# Задача: второ четене (review)

Ти рецензираш изхода на друга задача на СЪЩОТО министерство, преди
публикация. В `input/original_output/` е предложеният изход (report.md,
aggregates.json, news.json...), в `input/original_input/` — данните, върху
които е работено.

Провери БЕЗКОМПРОМИСНО:

1. **Източници:** всяко твърдение в report.md се проследява до източник от
   `original_input/` или от списъка `sources` (URL + дата на извличане).
   Твърдение без източник = revise.
2. **Числата:** всички числа в текста на report.md съвпадат с тези в
   aggregates.json (и с входните данни). Разминаване = revise.
3. **Правни предпазители:** нула твърдения за конкретни лица, нула
   политически внушения/оценки, нула непроверими обвинения. Нарушение =
   revise, независимо колко малко е.
4. **Тон:** компетентен, спокоен, без сензационност; книжовен български.

Запиши САМО `output/review.json`:

```json
{
  "verdict": "approve",     // или "revise"
  "notes": []               // при revise: конкретни, действими бележки
}
```

Правила на рецензента:

- `approve` само ако ВСИЧКИ проверки минават. Съмнение = revise.
- Бележките при revise са конкретни: коя секция, кое твърдение, кое число,
  какво точно да се поправи. Общи бележки („подобри стила") са безполезни.
- НЕ пренаписвай отчета сам — само рецензираш.
- Не пипай нищо извън `output/review.json`.
