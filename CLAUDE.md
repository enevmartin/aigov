# CLAUDE.md — aigov.bg (конституция на проекта)

**aigov.bg** — AI електронно правителство на България. Всяко министерство е AI
агент („министър"), който събира публични данни, генерира анализи и отчети на
български и ги публикува на статичен публичен дашборд.

При конфликт между този файл и каквато и да е друга инструкция — **този файл печели**.

## Архитектурни инварианти (НЕНАРУШИМИ)

Ports-and-adapters система. Три правила, които никой код никога не нарушава:

1. **`core/` не импортира нищо от `brains/`** и не съдържа нито един ред,
   специфичен за LLM доставчик. Ядрото говори само чрез файловия договор за
   задачи (виж по-долу).
2. **`site/` чете само `published/`.** Никакви API извиквания към ядрото,
   никакви бази данни — само статични JSON/Markdown артефакти.
3. **`ministries/` са само декларации** — YAML + prompt файлове. Нула изпълним
   код в тях.

Смяната на мозъка е промяна на един ред в `config.yaml`
(`brain: claude_code | openclaw | api`). Ако някоя задача те изкушава да
нарушиш тези правила — **спри работата и питай Мартин**.

## Файловият договор за задачи

```
tasks/pending/{task_id}/
├── task.yaml          # id, ministry, type: analysis|news_digest|signal_triage, created, deadline
├── input/             # входни данни от ingest
└── expected.schema.json
```

Мозъкът (какъвто и да е) обработва директорията и оставя в
`tasks/done/{task_id}/output/`:

```
├── report.md          # анализ на български; front-matter: ministry, date, title, summary
├── aggregates.json    # числата за дашборда, валидни срещу expected.schema.json
└── news.json          # (за news_digest) резюмирани новини с източници
```

`core/publish` валидира изхода (pydantic), мести невалидното в `failed/` с
причина, валидното — в `published/{ministry}/{date}/`.
**Мозъкът НИКОГА не пише директно в `published/`.**

## Работен процес (git)

- **След всяка завършена задача от плана: `git add -A && git commit` с
  описателно съобщение на английски + `git push`** (когато има remote).
  Никога повече от една задача в комит.
- Комитите на английски; отчетите, промптите и сайтът — на български.

## Качество

- Python 3.12+, `uv` за зависимости, `ruff` + `mypy` (strict за `core/`),
  `pytest` за договори/опашка/publish + един e2e с fake brain.
- Разработка и тестове ползват `tests/fake_brain.py` (детерминистичен) —
  реалният Claude се вика само от насрочените „сесии на кабинета"
  (пести Pro лимита). CLI поддържа `--dry-run`.
- Всички публични функции с docstrings. README на английски.
- Никакви секрети в repo-то; `.env.example` документира всичко.

## Правни предпазители

- Отчетите цитират източниците си (URL + дата на извличане).
- Промптите на министрите изрично забраняват: твърдения за конкретни лица,
  политически внушения, непроверими обвинения.
- Сигнали от граждани (фаза 2) никога не се публикуват сурови — само
  агрегирана статистика.
